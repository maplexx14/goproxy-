# Secure Proxy (VPN / Tunnel)

Простой, защищенный и устойчивый к блокировкам SOCKS5-прокси. Он обеспечивает маскировку трафика под обычный WebSocket (HTTPS), что усложняет его детектирование (DPI). Кроме того, сервер имеет встроенную защиту-«канарейку» против активного сканирования (active probing).

## Как это работает?
* **Клиент** создает локальный SOCKS5 прокси-сервер на вашем компьютере (по умолчанию 127.0.0.1:1080).
* **Сервер** устанавливается на удаленный VPS (линукс/windows).
* Весь заблокированный трафик направляется через WebSocket туннель к серверу.
* Известные отечественные сайты, добавленные в *White-лист* (`config.json`), открываются напрямую, минуя удаленный сервер! Это экономит трафик и ускоряет загрузку местных сервисов.

## 🚀 Установка и запуск (Сервер - на VPS)

1. Установите Go (`apt install golang-go` или по инструкции с оф. сайта).
2. Скопируйте файл `server.go` и скрипт `start_server.sh` на ваш сервер.
3. Выполните в терминале:
   ```bash
   chmod +x start_server.sh
   ./start_server.sh
   ```
4. Вы увидите сообщение с IP (который нужно узнать самому) и портом (по умолчанию `8080`).

## 💻 Установка и запуск (Клиент - на вашем ПК)

1. На Windows просто запустите `start_client.bat`. Он сам скомпилирует и запустит программу.
2. При первом запуске клиент спросит вас **IP и порт вашего VPS**.
3. Введите данные (например: `123.45.67.89:8080`) и нажмите Enter. Программа сохранит `config.json` и будет автоматически запускаться с этими настройками в следующий раз.

## ⚙️ Настройка браузера / Telegram

После запуска `start_client.bat` ваша программа готова. Не закрывайте черное окно.

**Для Telegram:**
1. Настройки -> Продвинутые настройки -> Тип соединения (Использовать собственный прокси)
2. Выберите SOCKS5
3. Хост: `127.0.0.1`
4. Порт: `1080` (или тот, что настроили в config.json)
5. Сохранить.

**Для Браузера (Chrome/Edge/Яндекс):**
Рекомендуется использовать расширения наподобие **Proxy SwitchyOmega** или **FoxyProxy**.
* Протокол: `SOCKS5`
* Хост: `127.0.0.1`
* Порт: `1080`

## 🛡️ Белый список (Whitelist)

В файле `config.json` на клиенте содержится массив `whitelist`.
Туда можно добавлять окончания доменов, которые **не должны** обходить блокировку. 
Например:
```json
"whitelist": [
  "yandex.ru",
  "vk.com",
  "mail.ru",
  "gosuslugi.ru",
  ".ru"
]
```
При добавлении сайтов в белый список, они всегда будут загружаться быстрее и напрямую через вашего провайдера.
